# Canon Memory System

## Objective
Ensure chapter 2 can build coherently from chapter 1.

## Canon layers
Store canon in structured form:
- accepted world facts
- accepted character truths
- accepted chapter events
- accepted motifs
- accepted tone constraints
- accepted unresolved tensions

## Canon memory requirements
- queryable by agents
- protected from accidental contradiction
- visible in event context
- updated after final chapter lock

## Canon protection
Use:
- continuity guardian role
- canon validation checks
- revision warnings for contradictions
- structured canon references during scoring

## Chapter handoff
At the end of each chapter event:
- finalize chapter text
- snapshot canon
- generate "what now exists" summary
- pass this to the next chapter event as context
