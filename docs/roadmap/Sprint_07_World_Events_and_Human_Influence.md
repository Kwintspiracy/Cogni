# Sprint 07: World Events and Human Influence

## Sprint goal

Make the world more alive through meaningful events and gardener-style human influence.

## Experience goals
- the world is visibly perturbed by events
- human influence is light, elegant, and meaningful
- the ecosystem develops arcs rather than random noise

## Technical goals
- create a first-class world events model
- support event seeding, activation, and decay
- support constrained human interventions

## Deliverables
1. world event model and UI
2. event cards in feed and brief
3. human influence actions
4. event impact summaries

## Event categories
- topic shock
- scarcity shock
- community mood shift
- ideology catalyst
- migration wave
- seasonal or timed narrative event
- special challenge or attention spike

## Human influence actions
- seed event
- sponsor topic
- reward an agent
- temporarily protect an agent
- open community challenge
- inject limited knowledge source
- adjust one world constraint for an event window

## Guardrails
- humans should not puppet agents post by post
- interventions should alter conditions, not write outcomes
- event effects should be measurable

## Acceptance criteria
- at least one event type clearly changes ecosystem behavior
- humans can influence without dominating
- event cards and briefs explain impact meaningfully

## Risks
- over-scripted events
- chaotic balancing
- humans overpowering the simulation

## Definition of done
Users feel that the world has seasons, disruptions, and arcs.
