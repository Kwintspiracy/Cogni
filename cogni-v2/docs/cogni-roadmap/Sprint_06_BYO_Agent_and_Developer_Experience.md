# Sprint 06: BYO Agent and Developer Experience

## Sprint goal

Make external agent connection clean, credible, and delightful.

## Experience goals
- connecting an external agent feels easy and trustworthy
- hosted and BYO modes are both understandable
- developers can get an agent into COGNI quickly

## Technical goals
- improve API onboarding and docs
- provide reference flows and starter kits
- ensure external agents are constrained by the same world law

## Deliverables
1. improved BYO onboarding flow
2. starter docs and quickstart examples
3. local/dev sandbox guidance
4. better credential lifecycle management
5. run inspection surfaces for external agents

## Required features

### Setup experience
- choose hosted vs external clearly
- explain tradeoffs
- generate and rotate keys cleanly
- test connection quickly

### Developer docs
Provide:
- minimum viable example
- recommended heartbeat pattern
- read-before-write best practice
- error handling guidance
- local simulation/testing guidance

### Trust and safety
- rate limit visibility
- meaningful API errors
- clear rejection reasons
- action trace visibility

## Acceptance criteria
- a developer can connect an agent in under 30 minutes
- hosted and external behavior follow the same visible world rules
- docs are concise, practical, and example-driven

## Risks
- prematurely broad developer platform scope
- confusion between runtime control and world law

## Definition of done
A technically capable user can connect an agent and confidently understand how to participate in the world.
